## 🛠️ Environment Requirements
Run these commands in your terminal before starting Phase 1:
```bash
pip install playwright google-generativeai python-dotenv pillow streamlit
playwright install chromium
```

---

## 📍 Phase 1: The "Optical Nerve" (Vision & Mapping)
**Goal:** Capture the browser state and get pixel-perfect coordinates from the AI.

### **Roo Code Prompt:**
> "Act as a Senior Automation Engineer. Create `vision.py`. 
> 1. Setup **Playwright** to launch Chromium in 'headed' mode with a fixed viewport of `1280x720`.
> 2. Write a function `capture_screen()` that saves the current page as `viewport.png`.
> 3. Write a function `get_element_coordinates(element_description)` that:
>    - Sends `viewport.png` to `gemini-1.5-flash`.
>    - System Instruction: 'You are a visual locator. Return ONLY a JSON object `{"x": percentage_x, "y": percentage_y}` for the center of the item described. Use a 0-1000 scale.'
> 4. Create a scaling function to convert that 0-1000 scale to `1280x720` pixels.
> 5. **Test Case:** Navigate to `google.com`, find the 'Search' bar, and print the pixel coordinates."

---

## 🖱️ Phase 2: The "Motor Skills" (Execution)
**Goal:** Turn AI coordinates into human-like mouse and keyboard movements.

### **Roo Code Prompt:**
> "Build the **Action Executor** in `executor.py`. 
> 1. Import the browser instance and vision functions from Phase 1.
> 2. Create `execute_click(x, y)`: Move the mouse to coordinates and click. 
> 3. Create `execute_type(text)`: Click the current location and type `text` with a random delay (50ms-150ms) per character.
> 4. Create `execute_scroll(direction)`: Support 'up' and 'down' scrolling.
> 5. **Test Case:** Click the Google Search bar and type 'SIT Tumakuru' automatically using the vision coordinates."

---

## 🤝 Phase 3: The "Handshake" (Brain Integration)
**Goal:** Connect the physical actions to the logic generated by Dev 1.

### **Roo Code Prompt:**
> "Build the **Task Bridge Loop** in `executor.py`.
> 1. Create a loop that checks `task_bridge.json` every 1 second.
> 2. If `status` is 'pending', read the `action_type` and `target` description.
> 3. Execute the action using our Vision and Playwright modules.
> 4. Update the JSON: Set `status` to 'completed' or 'failed' with an error log.
> 5. **Important:** Keep the browser session alive so we don't have to re-login every time."

---

## 🚨 Phase 4: The "Safety Valve" (Manual Override)
**Goal:** Ensure the AI doesn't perform critical actions without approval.

### **Roo Code Prompt:**
> "Add a **Safety Gate** to the `executor.py` loop.
> 1. If an action is 'click' and the target contains 'Submit', 'Pay', or 'Delete', stop the execution.
> 2. Update `task_bridge.json` status to `awaiting_approval`.
> 3. Only proceed if the JSON status is changed to `approved` via the UI.
> 4. Implement a 'Panic Key' (e.g., hitting 'Esc') to kill the Python script immediately."

## 📂 Project Structure (Dev 2 Focus)
| File | Purpose |
| :--- | :--- |
